# 日志系统开发文档

## 1. 概述

ShiJing 的日志模块位于 `Utils/Logger/`，命名空间 `ShiJing.Utils.Log`，是一套轻量、自包含、线程安全的本地日志组件。

设计目标：

- **轻量无依赖**：不引入第三方日志框架，全部基于 BCL 实现。
- **线程安全**：多线程并发写日志互不阻塞，单目标失败不影响其他目标。
- **可配置**：通过 `LoggerConfig` 控制等级、输出目标、文件目录与滚动策略，支持运行期热加载。
- **可扩展**：以 `LogTarget` 为抽象基类，可按需新增输出目标（如网络、数据库）。
- **与业务解耦**：对外仅暴露 `ILogger` 接口与 `LoggerService` 单例，内部实现细节（`Logger`、`RollingFileWriter`）对调用方不可见。

> 现状说明：日志模块已完成核心实现与单元测试，但尚未接入业务代码（`App.xaml.cs` / `MainWindow.xaml.cs` 仍为 WPF 模板）。

## 2. 架构与分层

```
调用方 (业务代码)
   │  ILogger.Info / Warn / Error ...
   ▼
Logger (internal)            —— 持有 Source，构造 LogEntry 并委派
   │  LoggerService.Dispatch(LogEntry)
   ▼
LoggerService (单例)         —— 等级过滤、快照目标列表、异常隔离分发
   │  target.Write(entry)
   ▼
LogTarget (抽象基类)
   ├── ConsoleLogTarget      —— 控制台着色输出
   └── FileLogTarget         —— 文件输出
         │
         ▼
       RollingFileWriter (internal)  —— 单文件写入 + 按大小滚动
```

各层可见性：

| 层 | 主要类型 | 可见性 |
| --- | --- | --- |
| 接口层 | `ILogger` | `public` |
| 实现层 | `Logger` | `internal` |
| 服务层 | `LoggerService` | `public` |
| 数据模型 | `LogEntry`、`LogLevel`、`LoggerConfig`、`LogOutputTarget` | `public` |
| 目标层 | `LogTarget`、`ConsoleLogTarget`、`FileLogTarget` | `public` |
| 基础设施 | `RollingFileWriter` | `internal` |

## 3. 核心文件一览

| 文件 | 类型 | 可见性 | 职责 |
| --- | --- | --- | --- |
| `LogLevel.cs` | `enum` | `public` | 日志等级（Trace~Fatal） |
| `ILogger.cs` | `interface` | `public` | 对外日志接口，屏蔽底层实现 |
| `Logger.cs` | `class` | `internal` | `ILogger` 默认实现，持有 Source，委派给 `LoggerService` |
| `LoggerService.cs` | `class` | `public` | 单例服务，管理配置/目标/分发，线程安全 |
| `LoggerConfig.cs` | `class` + `enum` | `public` | 日志配置 POCO + `LogOutputTarget` 输出目标标志位 |
| `LogEntry.cs` | `class` | `public` | 不可变的单条日志数据模型 + `Format()` 格式化 |
| `LogTarget.cs` | `abstract class` | `public` | 输出目标抽象基类（Write/Flush/Dispose） |
| `ConsoleLogTarget.cs` | `class` | `public` | 控制台目标，按等级着色 |
| `FileLogTarget.cs` | `class` | `public` | 文件目标，写总文件 + 分等级文件 |
| `RollingFileWriter.cs` | `class` | `internal` | 单文件滚动写入器，按大小切分 |

## 4. 日志等级

`LogLevel` 采用整数值定义，便于用比较运算做等级过滤：

| 等级 | 值 | 含义 | 典型场景 |
| --- | --- | --- | --- |
| `Trace` | 0 | 跟踪：最详细的诊断信息 | 开发调试，生产通常关闭 |
| `Debug` | 1 | 调试：开发阶段辅助信息 | 排查问题时的中间状态 |
| `Info` | 2 | 信息：常规运行流程（默认最低等级） | 应用启动、正常操作记录 |
| `Warn` | 3 | 警告：非预期但可恢复 | 资源接近上限、降级处理 |
| `Error` | 4 | 错误：运行时错误，影响部分功能 | 异常捕获、操作失败 |
| `Fatal` | 5 | 致命：严重错误，核心功能不可用 | 启动失败、不可恢复状态 |

等级过滤规则：`LoggerService.Dispatch` 中 `entry.Level < Config.MinLevel` 的日志被直接丢弃，不进入任何目标。

## 5. 配置（LoggerConfig）

`LoggerConfig` 为可变 POCO，所有字段在 `LoggerService` 启动时读取；运行期修改需显式调用 `LoggerService.ReloadConfig`。

| 属性 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `MinLevel` | `LogLevel` | `Info` | 允许输出的最低日志等级，低于该等级被丢弃 |
| `OutputTarget` | `LogOutputTarget` | `All` | 输出目标（控制台/文件/两者），标志位可组合 |
| `LogDirectory` | `string` | `{BaseDirectory}/logs` | 文件日志根目录，文件目标启用时使用 |
| `AllFileName` | `string` | `all.log` | 总日志文件名（记录所有等级） |
| `LevelFileNamePrefix` | `string` | `level` | 分等级文件名前缀，最终形如 `level-error.log` |
| `MaxFileSizeBytes` | `long` | `10 MB` | 单个日志文件最大字节数，超过则滚动切分 |
| `MaxRollingFiles` | `int` | `10` | 每个日志文件保留的最大滚动副本数 |
| `NewLine` | `string` | `Environment.NewLine` | 格式化行尾 |

`LogOutputTarget` 为 `[Flags]` 枚举，可按位组合：

| 值 | 含义 |
| --- | --- |
| `None = 0` | 不输出 |
| `Console = 1` | 输出到控制台 |
| `File = 2` | 输出到文件 |
| `All = Console \| File` | 同时输出到控制台与文件（默认） |

`LoggerService.BuildTargets` 依据 `HasFlag` 决定构建哪些目标：含 `Console` 则加入 `ConsoleLogTarget`，含 `File` 则加入 `FileLogTarget`。

## 6. 输出目标

### 6.1 控制台目标（ConsoleLogTarget）

- 调用 `LogEntry.Format()` 得到单行文本后输出。
- **按等级着色**（仅对交互式控制台生效，重定向到文件时为纯文本）：

  | 等级 | 颜色 |
  | --- | --- |
  | Trace | `DarkGray` |
  | Debug | `Gray` |
  | Info | `White` |
  | Warn | `Yellow` |
  | Error | `Red` |
  | Fatal | `Magenta` |

- **输出流分流**：`Warn` 及以上（`level >= LogLevel.Warn`）写入 `Console.Error`，其余写入 `Console.Out`。
- 输出前后保存/恢复 `Console.ForegroundColor`，避免颜色串扰。

### 6.2 文件目标（FileLogTarget）

同时写入两类文件，每个文件独立滚动：

- **总日志文件**（`all.log`）：所有等级统一写入，便于全量排查。
- **分等级日志文件**（`level-<level>.log`）：每个等级一个独立文件（`level-trace.log`、`level-error.log` 等），便于按等级定位问题。文件名中小写化等级名（`level.ToString().ToLowerInvariant()`）。

文件目录在 `FileLogTarget` 构造时按需创建（`Directory.CreateDirectory`）。

## 7. 文件滚动切分策略（RollingFileWriter）

`RollingFileWriter` 负责单个逻辑日志文件（如 `all.log` 或 `level-error.log`）的写入与按大小滚动，线程安全（内部 `_lock`）。

**滚动流程**（当前文件写入后超限触发）：

1. 关闭当前写入流。
2. 删除序号最大的副本（如 `all.log.{MaxRollingFiles}`）。
3. 从大到小依次上移：`all.log.i` → `all.log.i+1`。
4. 当前文件 `all.log` 改名为 `all.log.1`。
5. 新建同名空文件继续写入。

副本命名约定：`all.log.1`、`all.log.2` …，**序号越小越新**（`.1` 是最近一次切分前的内容）。

其他细节：

- **追加模式续写**：以 `FileMode.Append` 打开，应用重启后继续在同一文件续写，`_currentSize` 取 `stream.Length`。
- **UTF-8 无 BOM**：`new UTF8Encoding(false)`，避免滚动后多个副本拼接时出现重复 BOM。
- **写入即 Flush**：每次 `Write` 后调用 `_writer.Flush()`，保证日志及时落盘（崩溃不丢日志），代价是 IO 较频繁。
- **缓冲区**：`FileStream` 使用 4096 缓冲 + `FileOptions.SequentialScan`，适配顺序追加。
- **先滚动再写入**：仅当当前文件已有内容（`_currentSize > 0`）且写入后会超限时才滚动，避免首条日志就触发空滚动。

## 8. 日志格式（LogEntry.Format）

当前格式（单行）：

```
{Timestamp} [{LEVEL}] [{Source}] {Message}{Exception}
```

- `Timestamp`：本地时间，格式 `yyyy-MM-dd HH:mm:ss.fff`。
- `LEVEL`：等级名大写（`INFO`、`WARN` 等）。
- `Source`：来源标记，为空时整段 `[{Source}]` 省略。
- `Exception`：非空时以换行符附加完整异常信息（`ToString()`）。

示例：

```
2026-06-23 14:30:15.421 [INFO] [MainWindow] snapshot saved
2026-06-23 14:30:16.102 [ERROR] [Billing] charge failed
System.ArgumentException: bad arg
   at ...
```

## 9. 线程安全与并发

- **`LoggerService`**：用 `_lock` 保护 `_config` / `_targets` / `_loggers` / `_disposed`。`GetLogger` 在锁内复用/创建 logger；`Dispatch` 仅在锁内取目标列表快照，随后**释放锁**再遍历写入，避免多线程写日志相互阻塞。
- **`Dispatch` 先过滤后分发**：先做 `entry.Level < MinLevel` 判定，避免无谓的锁与 IO。
- **各 `LogTarget` 内部自保证线程安全**（如 `RollingFileWriter` 的 `_lock`、`ConsoleLogTarget` 依赖控制台写入的原子性）。
- **单目标异常隔离**：`Dispatch` 对每个 `target.Write` 包 `try/catch`，失败时兜底打印到 `Console.Error`（再失败则忽略），不影响其他目标与调用方。
- **Dispose 后静默丢弃**：`Dispatch` 在 `_disposed` 后直接返回，不再分发；`Flush` 在 Dispose 后也不抛异常。

## 10. 生命周期与热加载

- **单例**：`LoggerService.Instance` 通过 `Lazy<LoggerService>`（`ExecutionAndPublication`）实现首次访问时按默认配置初始化。
- **可测试构造**：`new LoggerService(LoggerConfig)` 允许注入自定义配置，便于单元测试。
- **Dispose 语义**：`Dispose(true)` 标记 `_disposed`、清空 `_targets`、释放各目标资源；`Dispose()` 调用后 `ReloadConfig` 抛 `ObjectDisposedException`，`Dispatch` / `Flush` 静默返回。
- **热加载 `ReloadConfig`**：
  1. 锁内校验非空、非 disposed，保存旧 `_targets`，替换 `_config` 并 `BuildTargets` 重建。
  2. **锁外**释放旧目标，避免与正在写入的线程争用。
  3. 已创建的 logger 实例继续有效，自动使用新配置（因其持有的 `LoggerService` 引用不变）。
- **`Flush`**：遍历当前目标快照调用 `Flush()`，强制刷新文件流缓冲。

## 11. 典型用法

> 业务代码尚未接入，以下为推荐写法示例。

获取 logger 并记录：

```csharp
using ShiJing.Utils.Log;

// 通过类型获取绑定到全名的 logger
private static readonly ILogger _log =
    LoggerService.Instance.GetLogger(typeof(MainWindow));

_log.Info("应用启动");
_log.Warn("磁盘空间不足，已降级为低清截图");

try { DoWork(); }
catch (Exception ex)
{
    _log.Error("处理失败", ex);
}
```

按字符串来源获取（相同来源复用同一实例）：

```csharp
var log = LoggerService.Instance.GetLogger("Billing");
var same = LoggerService.Instance.GetLogger("Billing");
// ReferenceEquals(log, same) == true

// 创建子来源 logger
var child = log.ForSource("Billing.Refund");
```

自定义配置初始化（替换默认单例行为可在启动早期调用）：

```csharp
LoggerService.Instance.ReloadConfig(new LoggerConfig
{
    MinLevel = LogLevel.Debug,
    OutputTarget = LogOutputTarget.All,
    LogDirectory = Path.Combine(AppContext.BaseDirectory, "logs"),
    MaxFileSizeBytes = 5L * 1024 * 1024,
    MaxRollingFiles = 5,
    NewLine = "\n"
});
```

空来源/空类型兜底：`GetLogger("")`、`GetLogger((string)null)`、`GetLogger((Type)null)` 均归一化为来源 `"Application"`。

## 12. 扩展点

新增输出目标（如网络上报、数据库）：

1. 继承 `LogTarget`，实现 `Write(LogEntry)`（可选重写 `Flush` / `Dispose(bool)`）。
2. 在 `LoggerService.BuildTargets` 中按配置加入新目标，或通过反射注入（测试中 `RecordingLogTarget` 即用此方式）。

未来可演进方向：

- **调用点信息**：在 `Format()` 中加入调用方行号/类名/方法名（可借助 `[CallerLineNumber]` / `[CallerMemberName]` / `[CallerFilePath]`，开销最小）。
- **结构化日志**：支持键值对字段，便于后续接入集中式日志系统。
- **异步写入队列**：当前为同步写入，高并发场景可引入后台队列降低调用方阻塞。
