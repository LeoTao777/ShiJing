# 创建自定义控件指南（WPF + iNKORE.UI.WPF.Modern）

本指南以本项目 `Views/Controls/FolderPicker`（文件夹选择控件：左侧显示路径文本 + 右侧"浏览"按钮）为案例，说明在 ShiJing 项目中创建一个自定义 `UserControl` 的完整流程、约定与常见模式。

> 适用范围：封装 UI + 交互逻辑的可复用控件。若需要可换皮肤/可模板化的"无外观控件"，应改用继承 `Control` + `ControlTemplate` 的方式，不在本指南范围。

---

## 一、整体思路

自定义控件 = **XAML 布局** + **code-behind 暴露依赖属性**。核心原则：

1. **控件不直接知道具体的 Model/ViewModel**——通过依赖属性对外暴露可绑定点，由使用方决定绑什么。
2. **内部事件用 code-behind 处理**（如按钮点击），不要为了"MVVM 纯洁"硬塞一个内部 ViewModel。
3. **对外用依赖属性（DependencyProperty）**，而非普通 CLR 属性，否则无法双向绑定、无法响应外部变更。

---

## 二、步骤

### 步骤 1：建文件

放在 `Views/Controls/` 下，文件名即控件名。两个文件：

- `FolderPicker.xaml` —— 根标签 `UserControl`
- `FolderPicker.xaml.cs` —— partial class 继承 `UserControl`

> 在解决方案资源管理器里"添加 → 用户控件(WPF)"会自动生成成对的 `.xaml` + `.xaml.cs`，且 `InitializeComponent()` 接线正确，推荐用这种方式新建，避免手写漏配。

### 步骤 2：写 XAML 布局

`FolderPicker.xaml`：

```xml
<UserControl
    x:Class="ShiJing.Views.Controls.FolderPicker"
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
    xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
    xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"
    xmlns:ui="http://schemas.inkore.net/lib/ui/wpf/modern"
    d:DesignHeight="40"
    d:DesignWidth="400"
    mc:Ignorable="d">
    <Grid>
        <Grid.ColumnDefinitions>
            <ColumnDefinition Width="*" />
            <ColumnDefinition Width="Auto" />
        </Grid.ColumnDefinitions>

        <!-- 路径显示文本，只读，超长时省略号；圆角 -->
        <Border
            Grid.Column="0"
            Margin="0,0,8,0"
            Padding="10,6"
            VerticalAlignment="Center"
            Background="{DynamicResource {x:Static SystemColors.WindowBrushKey}}"
            BorderBrush="{DynamicResource {x:Static SystemColors.ControlDarkBrushKey}}"
            BorderThickness="1"
            CornerRadius="6">
            <TextBlock
                x:Name="PathText"
                VerticalAlignment="Center"
                Text="{Binding SelectedPath, RelativeSource={RelativeSource AncestorType=UserControl}, Mode=OneWay}"
                TextTrimming="CharacterEllipsis"
                ToolTip="{Binding SelectedPath, RelativeSource={RelativeSource AncestorType=UserControl}, Mode=OneWay}">
                <TextBlock.Style>
                    <Style TargetType="TextBlock">
                        <Style.Triggers>
                            <!-- 外部置为 false 时文字置灰 -->
                            <DataTrigger Binding="{Binding IsPathEnabled, RelativeSource={RelativeSource AncestorType=UserControl}}" Value="False">
                                <Setter Property="Foreground" Value="{DynamicResource {x:Static SystemColors.GrayTextBrushKey}}" />
                            </DataTrigger>
                        </Style.Triggers>
                    </Style>
                </TextBlock.Style>
            </TextBlock>
        </Border>

        <!-- 浏览按钮：打开文件资源管理器选择目标文件夹 -->
        <Button
            Grid.Column="1"
            Click="BrowseButton_Click"
            Content="浏览" />
    </Grid>
</UserControl>
```

**要点：**

- `xmlns:ui="http://schemas.inkore.net/lib/ui/wpf/modern"`：要用 iNKORE 控件（`SettingsCard`/`ToggleSwitch`/`FontIcon` 等）时加这个命名空间。注意 **iNKORE 没有移植 `TextBox`/`Button`**，这些直接用 WPF 原生（无前缀）。
- 绑定控件自身依赖属性用 `RelativeSource AncestorType=UserControl`，不能用 `ElementName` 绑还没声明的属性。
- 颜色用 `{DynamicResource SystemColors.*}` 跟随系统主题；iNKORE 主题键见 `ui:ThemeKeys`。
- `d:DesignHeight/Width` 仅影响设计器预览，不影响运行。

### 步骤 3：在 code-behind 暴露依赖属性

`FolderPicker.xaml.cs`：

```csharp
using System.Windows;
using System.Windows.Controls;

namespace ShiJing.Views.Controls
{
    /// <summary>
    /// 文件夹选择控件：左侧文本框显示路径，右侧按钮打开文件资源管理器选择目标文件夹。
    /// 通过 <see cref="SelectedPath"/> 依赖属性双向绑定到宿主的数据源。
    /// </summary>
    public partial class FolderPicker : UserControl
    {
        // —— 依赖属性：对外暴露的可绑定点 ——
        // 1) 主数据：选中路径（双向）
        public string SelectedPath
        {
            get => (string)GetValue(SelectedPathProperty);
            set => SetValue(SelectedPathProperty, value);
        }

        public static readonly DependencyProperty SelectedPathProperty =
            DependencyProperty.Register(
                nameof(SelectedPath),
                typeof(string),
                typeof(FolderPicker),
                new FrameworkPropertyMetadata(
                    string.Empty,
                    FrameworkPropertyMetadataOptions.BindsTwoWayByDefault | FrameworkPropertyMetadataOptions.Journal));

        // 2) 配置：对话框标题
        public string DialogTitle { get => (string)GetValue(DialogTitleProperty); set => SetValue(DialogTitleProperty, value); }

        public static readonly DependencyProperty DialogTitleProperty =
            DependencyProperty.Register(nameof(DialogTitle), typeof(string), typeof(FolderPicker),
                new PropertyMetadata("选择文件夹"));

        // 3) 配置：路径文本是否可用（false 时置灰）
        public bool IsPathEnabled { get => (bool)GetValue(IsPathEnabledProperty); set => SetValue(IsPathEnabledProperty, value); }

        public static readonly DependencyProperty IsPathEnabledProperty =
            DependencyProperty.Register(nameof(IsPathEnabled), typeof(bool), typeof(FolderPicker),
                new FrameworkPropertyMetadata(true));

        public FolderPicker()
        {
            InitializeComponent();
        }

        // —— 内部交互：按钮点击 ——
        private void BrowseButton_Click(object sender, RoutedEventArgs e)
        {
            using var dialog = new System.Windows.Forms.FolderBrowserDialog
            {
                Description = string.IsNullOrEmpty(DialogTitle) ? "选择文件夹" : DialogTitle,
                ShowNewFolderButton = true,
            };

            if (!string.IsNullOrEmpty(SelectedPath) && System.IO.Directory.Exists(SelectedPath))
                dialog.SelectedPath = SelectedPath;

            var owner = Window.GetWindow(this);
            var result = owner != null
                ? dialog.ShowDialog(new Win32Window(owner))
                : dialog.ShowDialog();

            if (result == System.Windows.Forms.DialogResult.OK)
                SelectedPath = dialog.SelectedPath;
        }

        private sealed class Win32Window : System.Windows.Forms.IWin32Window
        {
            private readonly IntPtr _handle;
            public Win32Window(Window window) => _handle = new System.Windows.Interop.WindowInteropHelper(window).Handle;
            public IntPtr Handle => _handle;
        }
    }
}
```

## 三、依赖属性（DependencyProperty）写法要点

这是自定义控件最核心、最容易出错的部分。

### 标准模板

```csharp
public bool IsPathEnabled
{
    get => (bool)GetValue(IsPathEnabledProperty);
    set => SetValue(IsPathEnabledProperty, value);
}

public static readonly DependencyProperty IsPathEnabledProperty =
    DependencyProperty.Register(
        nameof(IsPathEnabled),          // 属性名（用 nameof 避免改名漏改）
        typeof(bool),                   // 属性类型
        typeof(FolderPicker),           // 所属控件类型
        new FrameworkPropertyMetadata(true));  // 默认值 + 选项
```

### 三个坑

1. **必须用 `DependencyProperty`，不能用普通 CLR 属性。** 普通 CLR 属性无法被 XAML 双向绑定，外部改值控件也不会响应。
2. **CLR 包装器里只写 `GetValue`/`SetValue`，不要加任何逻辑。** WPF 绑定引擎直接调 `GetValue`/`SetValue` 绕过包装器，你在包装器里写的校验/副作用不会被触发。
   - 需要在值变化时做事 → 用 `PropertyChangedCallback`。
   - 需要校验值 → 用 `ValidateValueCallback` 或 `CoerceValueCallback`。
3. **命名必须一致：** 字段名 = 属性名 + `Property`（`IsPathEnabled` ↔ `IsPathEnabledProperty`），否则 XAML 编译/绑定会出问题。

### `FrameworkPropertyMetadata` 常用选项

```csharp
new FrameworkPropertyMetadata(
    defaultValue,                              // 默认值
    FrameworkPropertyMetadataOptions.BindsTwoWayByDefault,  // 默认双向绑定
    new PropertyChangedCallback(OnChanged),    // 值变化回调
    new CoerceValueCallback(Coerce)));         // 强制转换回调
```

- `BindsTwoWayByDefault`：像 `SelectedPath` 这类需要回写数据源的属性用。
- `Journal`：配合导航日记保留状态，可按需。

## 四、何时用依赖属性 vs 普通属性

| 场景 | 用什么 |
|---|---|
| 需要被 XAML 绑定（尤其双向/触发器） | **依赖属性** |
| 仅控件内部状态、不对外 | 普通 CLR 属性 |
| 控件要对外发事件 | **路由事件**（`RoutedEvent`）或暴露 `DelegateCommand` |

> 本项目用了 Prism，VM 间通信用 `BindableBase`/`DelegateCommand`；但**控件内部**对 XAML 暴露一律用依赖属性，不要混用。

## 五、内部交互用 code-behind，不要塞内部 ViewModel

`FolderPicker` 的"点按钮 → 弹文件夹对话框 → 写回路径"是**控件自身职责**，直接在 `BrowseButton_Click` 里处理。理由：

- 控件应是自包含黑盒，使用方只管绑定 `SelectedPath`，不该关心内部怎么弹窗。
- 若为每个控件建内部 ViewModel，会导致 DataContext 嵌套混乱（控件内绑定自己的 VM，使用方又绑外部 VM）。

> 例外：控件交互很复杂、有独立状态机时，可用内部 VM，但需明确 DataContext 边界。本项目的简单控件不需要。

## 六、外部配置/状态联动的两种模式

### 模式 A：直接绑控件原生属性
如 `SettingsCard.IsEnabled="{Binding ...}"`——WPF 控件自带的依赖属性可直接绑。

### 模式 B：自定义依赖属性 + DataTrigger
如 `FolderPicker.IsPathEnabled`，外部绑 `IsEnabled`，内部用 `DataTrigger` 据此改外观：

```xml
<Style.Triggers>
    <DataTrigger Binding="{Binding IsPathEnabled, RelativeSource={RelativeSource AncestorType=UserControl}}" Value="False">
        <Setter Property="Foreground" Value="{DynamicResource {x:Static SystemColors.GrayTextBrushKey}}" />
    </DataTrigger>
</Style.Triggers>
```

> 为什么不直接绑 `TextBlock.IsEnabled`？因为 `IsEnabled=false` 会让控件整体灰显且禁用交互，而我们只想改文字颜色、保留可点。自定义属性 + 触发器更精确。

## 七、项目相关注意事项

1. **命名空间约定**：自定义控件放 `ShiJing.Views.Controls`，使用方加 `xmlns:controls="clr-namespace:ShiJing.Views.Controls"`。
2. **WinForms 互操作**：本项目启用了 `UseWindowsForms`（仅 `FolderBrowserDialog` 用），并已在 `.csproj` 用 `<Using Remove="System.Windows.Forms" />` 移除其隐式 using。因此在 code-behind 里**必须用全限定名** `System.Windows.Forms.FolderBrowserDialog`、`System.Windows.Forms.DialogResult` 等，不能 `using System.Windows.Forms;`，否则与 WPF 的 `UserControl`/`Button` 等同名类型歧义。
3. **构建验证**：改完 `dotnet build`，确保 0 错误；本项目主项目保持 0 警告。

## 八、快速 checklist

新建一个自定义控件时逐项核对：

- [ ] 文件放在 `Views/Controls/`，命名空间 `ShiJing.Views.Controls`
- [ ] `.xaml` 根标签 `UserControl`，含 `xmlns:ui`（如用 iNKORE 控件）
- [ ] 对外数据/配置项声明为**依赖属性**，字段名 = 属性名 + `Property`
- [ ] CLR 包装器只含 `GetValue`/`SetValue`，副作用放 `PropertyChangedCallback`
- [ ] 内部交互在 code-behind 处理，不建内部 VM
- [ ] 绑定自身属性用 `RelativeSource AncestorType=UserControl`
- [ ] 颜色用 `{DynamicResource SystemColors.*}` 或 iNKORE `ThemeKeys` 跟随主题
- [ ] `dotnet build` 0 错误 0 警告
- [ ] 在使用方加 `xmlns:controls` 并验证绑定生效
